# Design Support System - 자주 묻는 질문 (FAQ)
**최종 업데이트:** 2026-05-07

---

## 시작하기

### Q1. Design Support System은 어떤 서비스인가요?
A: 근거 기반 디자인 창작 지원 플랫폼입니다. 편집툴이나 AI 이미지 생성기가 아니라, 디자인 목적을 구조화하고 트렌드 근거로 컨셉을 결정하며, 레퍼런스를 추상화해 시각화하고, 스펙 문서로 남기는 시스템입니다.

### Q2. 어떤 도메인을 지원하나요?
A: 산업디자인, 패션디자인, 시각디자인, 광고디자인 4개 도메인을 지원합니다. 각 도메인은 공통 파이프라인 위에 특화된 분석, 시각화, 스펙 템플릿을 제공합니다.

### Q3. 기존 AI 이미지 생성기와 어떻게 다른가요?
A: 기존 도구는 "무엇을 만들지"에 집중하지만, 본 시스템은 "왜 이 컨셉인가"에 답합니다. 모든 컨셉 결정은 트렌드 근거와 레퍼런스 출처에 기반하며, 레퍼런스를 모방이 아닌 추상화 문법으로 변환합니다.

### Q4. 디자인 전문가가 아니어도 사용할 수 있나요?
A: 네. 챗봇 협업 모드에서는 AI가 단계별로 질문을 드리며 진행합니다. 자동 진행 모드에서는 목적만 입력하면 시스템이 끝까지 진행합니다.

---

## 워크플로우

### Q5. 챗봇 협업 모드와 자동 진행 모드의 차이는?
A: 챗봇 협업 모드는 디자이너와 AI가 단계별로 컨셉을 좁혀가는 모드입니다. 각 단계에서 사용자가 직접 결정합니다. 자동 진행 모드는 목적과 제약만 입력하면 시스템이 끝까지 진행하며, 자동 결정마다 점수, 근거, 대안을 저장합니다.

### Q6. 중간에 특정 단계부터 다시 시작할 수 있나요?
A: 네. 챗봇 협업 모드와 자동 진행 모드 모두 특정 단계로 돌아가 재실행이 가능합니다. 기존 결정은 Decision Log에 유지됩니다.

### Q7. 파이프라인은 몇 단계인가요?
A: 17단계입니다. 목적 입력, 브리프 구조화, 스케치 업로드(선택), 추가 질문, 트렌드 조사, 컨셉 후보 생성, 컨셉 평가, 컨셉 결정, 레퍼런스 검색, 레퍼런스 분석, 스케치 분석, 추상화, 시각화, 대상물 적용, 후보 비교, 스펙 문서 작성, 검토/승인 순서입니다.

## 사용자 스케치

### Q8. 어떤 스케치를 업로드할 수 있나요?
A: 러프 스케치(손그림, 낙서), 구조 스케치(제품 단면, 실루엣), 스타일 스케치(패션 룩, 그래픽 레이아웃), 혼합 자료(스케치+메모+레퍼런스) 모두 가능합니다.

### Q9. 업로드한 스케치 원본이 수정되나요?
A: 아니요. 원본 스케치는 절대 덮어쓰지 않습니다. 구체화 결과는 별도 버전으로 저장됩니다.

### Q10. AI가 스케치를 잘못 해석하면 어떻게 하나요?
A: AI 해석은 "확정"이 아니라 사용자 확인이 필요한 가설로 표시됩니다. Sketch Input Board에서 해석을 수정하고 유지할 요소를 직접 선택할 수 있습니다.

## 레퍼런스

### Q11. 레퍼런스를 그대로 복제하나요?
A: 아니요. 레퍼런스는 모방 대상이 아니라 분석 대상입니다. 시스템은 레퍼런스를 형태, 구조, 비례, 재질, 상징, 사용성 규칙으로 변환합니다.

### Q12. 저작권 위험은 어떻게 관리되나요?
A: 모든 레퍼런스에 출처, 라이선스, 저작권 위험이 표시됩니다. 위험이 높은 자료는 추상화 전용으로만 사용되며 직접 스타일 적용은 제한됩니다.

### Q13. 출처 없는 결과가 나오나요?
A: 출처 없는 트렌드 주장은 컨셉 결정의 근거로 사용하지 않습니다. 출처가 불확실한 내용은 "가설" 또는 "검증 필요"로 명확히 표시됩니다.

---

## 추상화

### Q14. 추상화란 무엇인가요?
A: 레퍼런스를 디자인 문법으로 바꾸는 과정입니다. 형태, 구조, 표면, 색/재료, 의미, 사용성 6개 축으로 분석하여, 관찰한 요소를 실제 디자인 대상에 적용하는 규칙을 만듭니다.

### Q15. 특정 브랜드/작가 스타일을 따라할 수 있나요?
A: 아니요. 특정 브랜드나 작가의 스타일을 그대로 모사하지 않습니다. 레퍼런스의 디자인 문법만 추출하여 추상화 규칙으로 변환합니다.

---

## 컨셉과 결정

### Q16. 컨셉 후보는 어떻게 평가되나요?
A: 각 후보마다 트렌드 근거, 레퍼런스 가능성, 제작 리스크를 기반으로 점수가 매겨집니다. 점수와 함께 근거 출처와 리스크가 카드에 표시됩니다.

### Q17. 선택하지 않은 컨셉은 어떻게 되나요?
A: 보류하거나 폐기한 컨셉도 Decision Log에 기록됩니다. 스펙 문서에는 버린 대안과 선택 사유도 포함됩니다.

### Q18. 결정을 번복할 수 있나요?
A: 네. Decision Log에서 이전 결정을 확인하고, 언제든 다른 컨셉으로 돌아갈 수 있습니다. 변경 이력은 버전으로 저장됩니다.

---

## 스펙 문서

### Q19. 스펙 문서에 무엇이 포함되나요?
A: 프로젝트 브리프, 트렌드 근거, 컨셉 후보와 평가, 최종 컨셉 결정, 사용자 스케치 원본과 AI 해석, 레퍼런스 보드, 추상화 규칙, 스케치와 생성 이미지, 최종안 비교, 도메인별 스펙, 출처/라이선스/AI 사용 고지가 포함됩니다.

### Q20. 스펙 문서를 버전 관리할 수 있나요?
A: 네. 스펙 문서는 버전 관리가 가능하며, 승인 워크플로우를 통해 팀과 클라이언트가 검토할 수 있습니다.

## AI 모델

### Q21. 어떤 AI 모델을 사용하나요?
A: 기능별로 다른 모델을 사용합니다. 트렌드 조사, 컨셉 챗, 스케치 분석, 레퍼런스 분석, 추상화, 이미지 생성, 스펙 작성, 검증 각각에 최적화된 모델이 관리자 페이지에서 설정됩니다.

### Q22. 모델을 직접 선택할 수 있나요?
A: 관리자 콘솔에서 기능별 모델 정책을 설정할 수 있습니다. Provider, 모델명, 파라미터, Fallback 정책을 관리합니다. 코드에 모델명이 하드코딩되지 않습니다.

## 보안과 데이터

### Q23. 데이터는 안전한가요?
A: 모든 데이터는 Tenant와 Workspace에 종속되어 관리됩니다. SaaS 멀티테넌시 구조로 다른 조직의 데이터에 접근할 수 없습니다. 관리자 작업은 AuditLog에 기록됩니다.

### Q24. 팀원과 공유할 수 있나요?
A: 네. 워크스페이스 내에서 프로젝트와 세션을 공유할 수 있으며, 스펙 문서에 대한 승인 워크플로우를 지원합니다.
