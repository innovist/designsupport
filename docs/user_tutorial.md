# Design Support System - 사용자 튜토리얼
**Version:** 2.0.0
**업데이트:** 2026-05-07

---

## 1. 시작하기

Design Support System에 오신 것을 환영합니다. 본 시스템은 디자이너의 창작 과정을 지원하는 근거 기반 디자인 발상 도구입니다. 편집툴이나 AI 이미지 생성기가 아니라, 디자인 목적을 구조화하고 트렌드 근거로 컨셉을 결정하며, 스펙 문서로 남기는 플랫폼입니다.

### 1.1. 시스템 요구사항
- **웹 브라우저**: Chrome, Firefox, Safari, Edge 최신 버전
- **인터넷**: 안정적인 고속 인터넷 연결
- **해상도**: 권장 1920x1080 이상

## 2. 워크스페이스 둘러보기

로그인 후 나타나는 화면은 세 영역으로 구성됩니다.

```
+--------------------------------------------------------------------------------+
| Workspace / Project / Session / Mode: Guided or Auto                            |
+----------------------+--------------------------------------+------------------+
| Project Navigator    | Design Studio                         | Decision Panel   |
| - Project list       | [Chat with Design Expert]             | Current Stage    |
| - Session timeline   | [Evidence Board]                      | Brief Score      |
| - User sketches      | [Reference Clusters]                  | Concept Scores   |
| - Reference library  | [Sketch Analysis]                     | Selected Sketch  |
| - Generated assets   | [Abstraction Sketches]                | Selected Refs    |
| - Spec documents     | [Design Variations]                   | Next Decision    |
+----------------------+--------------------------------------+------------------+
```

- **Project Navigator**: 프로젝트와 세션의 맥락을 잃지 않게 합니다.
- **Design Studio**: 대화, 근거, 시각화가 연결되는 중심 화면입니다.
- **Decision Panel**: 다음 행동과 결정 근거를 압축해서 보여줍니다.

## 3. 워크플로우: 챗봇 협업 모드

### 3.1. 목적 입력과 브리프 구조화

1. 새 세션 생성 후 도메인을 선택합니다 (산업/패션/시각/광고)
2. 디자인 목적을 자연어로 입력합니다
   - 예: "휴대폰 거치대를 자연물 컨셉으로 디자인하고 싶다"
3. 시스템이 목적을 분석해 구조화된 브리프를 생성합니다
4. AI가 누락된 정보에 대한 추가 질문을 합니다

### 3.2. 사용자 스케치 업로드 (선택)

1. Sketch Input Board에서 스케치를 업로드합니다
2. 스케치 유형을 선택합니다: 러프 / 구조 / 스타일 / 혼합
3. 스케치에 대한 간단한 설명을 추가합니다
4. AI가 스케치의 의도, 형태, 구조를 분석합니다
5. 분석 결과를 확인하고 승인/수정합니다

**주의**: 원본 스케치는 절대 덮어쓰지 않습니다. AI 해석은 "확정"이 아니라 가설로 표시됩니다.

### 3.3. 트렌드 근거 조사

1. AI가 브리프 기반으로 트렌드 자료를 수집합니다
2. Evidence Board에서 출처가 포함된 인사이트를 확인합니다
3. 각 인사이트는 출처 URL, 발행일, 신뢰도가 표시됩니다
4. 출처가 불확실한 내용은 "가설" 또는 "검증 필요"로 표시됩니다

### 3.4. 컨셉 후보 생성과 결정

1. AI가 트렌드 근거를 바탕으로 컨셉 후보를 제시합니다
2. 각 후보 카드에 점수, 근거, 리스크가 표시됩니다
3. 선택 버튼으로 결정을 기록합니다: 채택 / 보류 / 폐기 / 더 탐색
4. 선택한 이유를 입력하면 Decision Log에 저장됩니다

### 3.5. 레퍼런스 검색과 선택

1. 선택한 컨셉을 기반으로 레퍼런스를 검색합니다
2. Reference Board에서 이미지/문서/사례를 확인합니다
3. 각 레퍼런스 카드에 출처, 라이선스, 저작권 위험이 표시됩니다
4. 사용할 레퍼런스를 선택하고, 제외하는 경우 사유를 기록합니다

### 3.6. 추상화와 시각화

1. AI가 선택된 레퍼런스를 디자인 문법으로 추상화합니다
2. Abstraction Board에서 형태/구조/표면/의미 규칙을 확인합니다
3. 스케치가 있는 경우 원본 방향을 유지한 구체화안이 생성됩니다
4. 다른 컨셉으로 확장한 변형안도 생성됩니다
5. 마음에 드는 방향을 선택합니다

### 3.7. 스펙 문서 생성

1. 모든 단계의 결과가 스펙 문서로 구조화됩니다
2. 문서에는 브리프, 트렌드 근거, 컨셉 후보, 결정 사유, 레퍼런스, 추상화 규칙, 최종안이 포함됩니다
3. 버전 관리가 가능하며, 승인 워크플로우를 지원합니다

## 4. 워크플로우: 자동 진행 모드

### 4.1. 사용법

1. 세션 생성 시 mode를 "auto"로 설정합니다
2. 목적과 제약만 입력합니다
3. 시스템이 끝까지 자동 진행합니다

### 4.2. 자동 모드 상태

| 상태 | 의미 | UI 표시 |
|---|---|---|
| queued | 작업 대기 | 예상 단계와 모델 표시 |
| researching | 트렌드/자료 조사 | 수집 출처와 진행률 |
| concepting | 컨셉 후보 생성 | 후보 생성 중 스켈레톤 |
| referencing | 레퍼런스 수집 | 출처별 수집 수 |
| abstracting | 디자인 문법 추출 | 형태/구조/의미 축 표시 |
| generating | 스케치/이미지 생성 | 생성 작업 로그 |
| documenting | 스펙 작성 | 문서 섹션 진행 |
| review_ready | 검토 가능 | 검토 필요 항목 강조 |
| failed | 실패 | 실패 단계, 원인, 재시도 조건 |

### 4.3. 자동 모드 특징

- 자동 결정마다 점수, 근거, 대안, 리스크가 저장됩니다
- 불확실성이 큰 항목은 "검토 필요"로 표시됩니다
- 특정 단계부터 재실행이 가능합니다
- 중간 과정과 근거를 숨기지 않습니다

## 5. 레퍼런스 검색기 사용법

### 5.1. 검색 유형

- **키워드 검색**: 컨셉, 도메인, 소재, 시대, 무드
- **이미지 검색**: 유사한 이미지 사례
- **스케치 기반 검색**: 사용자 스케치의 형태/구조와 연결되는 사례
- **문서 검색**: 트렌드 문서, 브랜드 문서
- **내부 자산 검색**: 이전 프로젝트, 팀 라이브러리

### 5.2. 레퍼런스 분류

| 분류 | 설명 |
|---|---|
| Nature | 자연물, 생물, 지형, 계절 |
| Product | 제품, 가구, 도구, 패키지 |
| Architecture | 건축, 공간, 구조, 파사드 |
| Fashion | 룩, 아이템, 소재, 실루엣 |
| Graphic | 포스터, 로고, 타이포, 패턴 |
| Advertising | 캠페인, 컷, 메시지, 매체 |
| Material | 소재, 표면, 질감, CMF |

### 5.3. 주의 사항

- 레퍼런스는 모방 대상이 아니라 분석 대상입니다
- 저작권 위험이 높은 자료는 추상화 전용으로만 사용합니다
- 모든 레퍼런스는 출처와 라이선스가 기록됩니다

## 6. 사용자 스케치 기능

### 6.1. 업로드 유형

| 유형 | 예 | 처리 방식 |
|---|---|---|
| 러프 스케치 | 손그림, 낙서, 아이디어 메모 | 형태와 의도 파악 |
| 구조 스케치 | 제품 단면, 조립 방식, 실루엣 | 기능 구조 분석 |
| 스타일 스케치 | 패션 룩, 그래픽 레이아웃, 광고 컷 | 분위기, 비례 분석 |
| 혼합 자료 | 스케치+메모+레퍼런스 이미지 | 사용자 아이디어와 외부 참고 분리 |

### 6.2. 핵심 원칙

- 원본 스케치를 절대 덮어쓰지 않습니다
- AI 해석은 사용자 확인이 필요한 가설입니다
- 구체화 결과는 원본의 어떤 요소를 유지/변형했는지 설명합니다

## 7. 추상화 엔진 이해하기

### 7.1. 추상화 축

| 축 | 질문 | 예시 |
|---|---|---|
| 형태 | 외곽선, 비례, 곡률 | 삼각 실루엣 |
| 구조 | 하중, 결합, 지지 | 경사 지지 |
| 표면 | 질감, 광택 | 무광 세라믹 |
| 색/재료 | 색상 대비, 소재 | 흙색, 암석색 |
| 의미 | 상징, 감정 | 안정감, 자연 |
| 사용성 | 기능과의 연결 | 세워짐, 접힘 |

### 7.2. 작동 방식

레퍼런스를 추상화 축으로 분석해 디자인 문법(규칙)으로 변환합니다. 이 규칙을 바탕으로 스케치와 변형 이미지를 생성합니다. 원본 레퍼런스의 구도를 그대로 복제하지 않습니다.

- 시각화: 형태 스케치, 구조 스케치, 사용 장면, 변형안
- 스펙: 치수 가정, 소재, 구조, 제조 방식, 사용 시나리오

### 8.2. 패션디자인
- 분석: 시즌, 타깃, 실루엣, 소재, 패턴, 착장
- 시각화: 무드보드, 룩 스케치, 착장 이미지, 패턴 방향
- 스펙: 아이템, 소재, 컬러, 패턴, 스타일링, 시즌 근거

### 8.3. 시각디자인
- 분석: 브랜드 톤, 색, 타입, 그리드, 레이아웃
- 시각화: 키비주얼, 로고 방향, 포스터, 그래픽 시스템
- 스펙: 색/타입/그리드, 사용 규칙, 금지 규칙, 매체 확장

### 8.4. 광고디자인
- 분석: 타깃 인사이트, 메시지, 후킹, 채널
- 시각화: 캠페인 컷, 소셜 소재, 스토리보드
- 스펙: 메시지, 채널, 비주얼 톤, 카피, 규격, CTA

## 9. 문제 해결

### Q: 트렌드 자료가 부족합니다
A: "근거 부족" 표시가 나오면 검색어를 확장하거나 도메인을 전환하세요. 수동으로 출처를 추가할 수도 있습니다.

### Q: 레퍼런스 검색 결과가 없습니다
A: 검색어를 구체화하거나, 카테고리를 변경하거나, 직접 이미지를 업로드하세요.

### Q: 라이선스 위험 표시가 있습니다
A: 해당 레퍼런스는 추상화 전용으로만 사용할 수 있습니다. 직접 스타일 적용은 제한됩니다.

### Q: AI가 스케치를 잘못 해석했습니다
A: AI 해석은 가설입니다. Sketch Input Board에서 해석을 수정하고 유지할 요소와 바꿀 요소를 직접 선택하세요.

### Q: 자동 모드 결과가 불만족스럽습니다
A: 특정 단계부터 재실행이 가능합니다. "검토 필요"로 표시된 항목을 먼저 확인하세요.
