# Plan 25: 비정상 종료 이후 전체 파이프라인/사용자 의도 정합성 검증

**작성일**: 2025-12-23 12:05  
**목적**: 크롤링→AI 분석→보고서→프롬프트→의상 이미지→블루프린트→모델 착장 이미지까지 전 흐름이 실제 데이터 기반으로 자동 동작하는지 재검증하고, 사용자 의도(숨은 의견·스토리 반영)에 부합하는지 확인한다.

---

## 0. 전제/제약
- 파일 ≤ 300 LOC, 함수 ≤ 50 LOC, 매개변수 ≤ 5, 순환 복잡도 ≤ 10.
- 변경 전 관련 파일 전체 읽기(부분 읽기 금지).
- 실제 실행 로그 기반 테스트 필수(가짜 데이터 금지).
- 백엔드/프론트엔드 동시 점검.

## 1. 핵심 구성 요소/기대 기능
- 크롤링: 실제 웹/커뮤니티/미디어 데이터 수집, 소스 누락/미등록 없음.
- 분석: 댓글/전사/본문에서 숨은 의견·잠재 니즈 추출.
- 보고서: 실제 데이터 기반 JSON 보고서 생성/저장.
- 이미지: 의상 디자인 + 모델 착장 이미지 생성.
- 블루프린트: 스케치/레이아웃/패턴 3종 생성.

## 2. 엣지 케이스
- Gemini 할당량 초과(429)로 분석/프롬프트 실패.
- 댓글 수집 0건 → 숨은 의견 분석 불가.
- 크롤러 설정에 있으나 서비스 미등록.
- SearXNG 미설정 상태 선택.
- STT 비활성화/실패로 전사 누락.

## 3. 구현/검증 단계
1) 로그/DB 최근 세션 결과로 전체 흐름 확인(완료/실패 원인 포함).
2) 데이터 흐름 추적(수집→분석→보고서→이미지→블루프린트→모델).
3) 실제 파이프라인 실행 테스트(네트워크/API 키 필요)로 재현 검증.
4) 댓글/전사 기반 숨은 의견 반영 여부 확인.
5) 프론트 UI에서 결과 노출 상태 점검.

## 4. 완료 기준
- 실제 실행 로그로 7단계 파이프라인 성공.
- 보고서/이미지/블루프린트/모델 이미지 모두 생성 확인.
- 댓글/전사 기반 인사이트가 분석/보고서에 반영됨.
- UI에서 결과가 누락 없이 표시됨.
