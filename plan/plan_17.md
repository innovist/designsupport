# Plan 17: 크롤링 실패 원인 분석, 로그 가시화, 진행률 정밀화, SearXNG 통합

**작성일**: 2025-12-23 01:06  
**목적**: 크롤링 실패/무한 진행 원인을 로그·DB·파이프라인 기준으로 규명하고, 개요 탭에 크롤링 로그/키워드/정밀 진행률을 표시하며, SearXNG를 통합해 트렌드 분석 데이터 품질을 개선한다.

---

## 0. 전제/제약
- Git 저장소 부재로 브랜치/PR 단계 수행 불가.
- 파일 ≤ 300 LOC, 함수 ≤ 50 LOC, 매개변수 ≤ 5, 순환 복잡도 ≤ 10 준수.
- 변경 전 관련 파일 전체 읽기(부분 읽기 금지).
- 분리 작업은 복잡도를 높이지 않도록 최소화.

## 1. 핵심 구성 요소 및 기대 기능
- 세션 실패 원인: 최근 로그/DB 기록 기반으로 실패 지점과 에러 확인.
- 워커 상태: 현재 수행 중 작업과 종료 조건 확인.
- STT 동작 여부: YouTube 수집 경로에서 STT 호출/결과 저장 확인.
- 크롤러 설정 충돌: 모달 설정 vs 설정 페이지 적용 범위 확인 및 단일화.
- 개요 탭 로그/키워드/진행률 표시: 실시간 로그 및 키워드, 정밀한 진행률 표시.
- SearXNG 통합: 검색 결과 수집 → 분석 입력으로 사용.

## 2. 기술 문서 리서치/검색
- SearXNG 공식 API 문서/응답 포맷 확인.
- 프로젝트 내 크롤러 파이프라인 문서/레퍼런스 확인: `crawlers/*`, `app/services/pipeline_orchestrator.py`.
- 로그/세션 저장 구조 확인: `app/api/session_store.py`, `storage/fashion.db`.

## 3. 엣지 케이스
- 크롤러 미선택 시 진행률/로그 출력.
- 세션 실패 시 오류 메시지 부재 문제(로그 누락).
- 수집량 설정이 0/1일 때 진행률 계산.
- 키워드 생성 실패/빈 키워드 목록.
- SearXNG 응답 실패/빈 결과.
- STT 실패 또는 자막 미제공 영상.

## 4. 데이터 흐름/영향도 분석
- 세션 생성 → 키워드 생성 → 크롤링 시작 → 로그/진행률 업데이트 → 분석 입력 저장.
- crawler_config 입력이 어디에서 최종적으로 해석되는지 호출 체인 추적.
- DB 저장 경로(세션 메타/로그/크롤링 결과) 확인.

## 5. 구현 단계(백엔드/프론트 동시)
1) 최근 로그·DB 확인으로 실패 지점/메시지 확보.
2) 크롤러 설정 충돌 원인 분석(모달 vs 설정 페이지) 및 단일화.
3) 크롤링 로그 저장/조회 API 확인 후 개요 탭에 터미널 UI 추가.
4) 진행률 계산 로직 개선(총 목표/완료 수 기반).
5) 개요 탭에 크롤링 키워드 표시.
6) SearXNG 수집 모듈 추가 및 파이프라인 연결.
7) 정적 검증 3회 + 전체 정적 검증 5회 실행.

## 6. 테스트 계획
- 실제 로그 기반 상태/오류 확인.
- 개요 탭 UI에서 로그/키워드/진행률 표시 확인.
- SearXNG 수집 결과가 분석 입력에 포함되는지 확인.
- 정적 검증 3회(phase) + 5회(전체) 실행 로그 확보.

## 7. 완료 기준
- 실패 원인/워커 동작/수집 대상이 로그로 명확히 파악됨.
- 진행률이 수집량 설정과 일치하고 하드코딩 제거됨.
- 개요 탭에 크롤링 로그(터미널 UI)와 키워드가 표시됨.
- SearXNG 수집이 분석 입력에 통합됨.
