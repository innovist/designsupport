# Plan 03 - 2026-05-10

## 목적
외부 크롤링 구현, UX/UI, 전체 파이프라인 정합성, 정적 검증, 런타임 E2E를 실제 동작 기준으로 검증하고 필요한 최소 수정만 수행한다.

## 가정
- 현재 `.env`의 외부 크롤러 API 설정을 사용한다.
- 실제 AI/이미지 생성 API는 설정된 제공자와 사용량 제한에 따라 실패할 수 있으므로, 실패 시 원인과 경로를 기록한다.
- 검증 중 발견한 이슈는 전체 검증 후 묶어서 수정한다.

## 단계별 작업 → 검증 방법
1. 크롤링 백엔드 단독 검증 → sources/start/status/data API 및 SearchClient 결과 확인.
2. UX/UI 사용성 검증 → 서버 실행 후 주요 페이지와 세션 상세 흐름을 브라우저/HTTP로 점검.
3. 정적 검증 → py_compile, pytest, JS 구문 검사, 주요 경로 검색.
4. 런타임 E2E → 프로젝트 생성, 세션 생성, 브리프 저장, 자동 파이프라인 실행, 진행/결과 DB·API 확인.
5. 이슈 일괄 수정 → 원인 기반 최소 패치 후 관련 검증 재실행.
