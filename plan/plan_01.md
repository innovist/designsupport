# 계획서 (plan_01.md)
작성일시: 2025-12-20 19:07:46 KST
버전: v0.1

## 목표/범위
- 트렌드 데이터 수집 → AI 분석/집계 → 디자인 콘셉트/보고서 → 프롬프트 → 이미지/도면 생성까지의 end-to-end 설계를 확정한다.
- 웹/앱 확장 가능 구조, 모듈화/클래스 구조/ORM 적용을 전제한다.

## 핵심 구성 요소 점검 (필수 작업 순서 1)
- 데이터 수집: 크롤러, 스케줄러, 출처/메타 저장
- 분석: 멀티 모델 분석 및 glm-4.7 최종 집계
- 디자인/프롬프트: 3개 콘셉트 도출, 초상세 프롬프트 생성
- 이미지 생성: 의상 전/후면, 착장, 다양한 배경/포즈, 도면 초안
- 일관성: 디자인 ID/레퍼런스 이미지/스타일 토큰 관리
- UI/UX: 입력/검토/선택/재생성, 이력/버전 관리

## 기술 문서 리서치 (필수 작업 순서 2,4,6)
- 크롤링 준수: RFC 9309 Robots Exclusion Protocol
- AI 리스크 관리: NIST AI Risk Management Framework
- 개인정보 보호: GDPR Regulation (EU) 2016/679

## 엣지 케이스 도출 (필수 작업 순서 3)
- 데이터 부족/편향, robots.txt 차단, 모델 간 의견 상충, 이미지 일관성 실패, 표준 치수 불일치

## 전체 구조/아키텍처
- 수집 계층: 소스 커넥터(사이트별 모듈), 스케줄러, 큐
- 처리 계층: 정제/중복 제거/신뢰도 점수화
- 분석 계층: 모델별 분석기 + 집계기(glm-4.7)
- 생성 계층: 프롬프트 생성기, 이미지 생성기, 도면 생성기
- 저장 계층: 원문/메타/분석 결과/이미지/도면/리포트 버전 관리
- 인터페이스 계층: 웹 UI + API, 향후 앱 확장 고려

## 데이터 흐름(요약)
- 입력(사용자 프롬프트/필터) → 크롤링 키워드 → 데이터 수집 → 정제/저장
- 모델 분석(3개) → 의견 집계(glm-4.7) → 보고서/콘셉트
- 보고서 → 초상세 프롬프트 → 이미지/도면 생성 → 결과 묶음(버전)

## 데이터 모델/스키마(초안)
- Source, CrawlJob, RawItem, TrendInsight, DesignConcept, PromptSpec, ImageAsset, PatternDraft, Report
