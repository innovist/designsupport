# Plan 13: UI 전체 i18n 누락 제거

**작성일**: 2025-12-22 21:08
**목적**: 모든 페이지/버튼/모달/기능 텍스트가 언어 변경에 따라 완전히 번역되도록 누락된 i18n 적용을 완료한다.

---

## 1. 범위 및 현황 점검
- 템플릿(templates/pages/*, base.html)과 페이지 스크립트(static/js/pages/*)의 하드코딩 문자열 전수 점검
- 동적 텍스트(크롤러 이름/카테고리, 유튜브 채널 UI, 알림/경고 메시지)의 번역 적용 여부 확인
- 백엔드 응답 문자열(에러 detail 등) 노출 지점 확인 후 프론트에서 번역 가능 여부 평가

## 2. 설계 방향
- **UI 텍스트는 i18n 키로만 렌더링**: data-i18n/data-i18n-placeholder 기반으로 통일
- **동적 데이터 라벨링**: 크롤러 id → i18n 키 매핑으로 이름/카테고리 번역
- **언어 변경 시 재렌더링 보장**: 언어 변경 이벤트에서 화면 재렌더링/데이터 재적용
- **누락 문자열 최소화**: tooltip/title/alert/confirm/prompt 메시지까지 i18n 적용

## 3. 변경 대상
- 템플릿: `templates/base.html`, `templates/pages/*.html`
- 페이지 스크립트: `static/js/pages/*.js`
- i18n 리소스: `static/i18n/ko.json`, `static/i18n/en.json`, `static/i18n/zh-CN.json`, `static/i18n/zh-TW.json`

## 4. 구현 단계
1) 템플릿 내 누락 텍스트에 data-i18n 및 관련 키 추가
2) JS 알림/경고/확인/프롬프트 문자열을 i18n 키로 교체
3) 크롤러/유튜브 채널 UI 동적 문자열의 번역 매핑 함수 추가
4) 언어 변경 이벤트에서 데이터 재적용 범위 확장
5) i18n JSON 4개 언어에 신규 키 추가 및 구조 정합화

## 5. 테스트 계획
- i18n 키 누락/하드코딩 텍스트 스캔 스크립트 실행(정적 점검)
- 언어 전환 이벤트 후 UI 재렌더링 확인(핵심 페이지: dashboard, settings, home, history, project_detail, chatbot)

## 6. 리스크
- 동적 데이터(크롤러/채널명) 번역 누락 시 UI 혼합 언어 발생 가능
- 백엔드 error.detail 노출 시 혼합 언어 유지 가능 → 프론트 메시지 우선 적용

## 7. 완료 기준
- 모든 페이지/버튼/모달/알림 문구가 i18n 키로 렌더링됨
- 언어 변경 시 UI 전체가 일관된 언어로 표시됨
- i18n JSON 4개 언어가 동일한 키 구조를 유지
