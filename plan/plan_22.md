# Plan 22: 사용자 의도 정합성 검증 및 자동 파이프라인 완결성 보강

**작성일**: 2025-12-23 04:20  
**목적**: 실제 데이터 기반 인사이트/숨은 의견 반영 여부와 크롤링→분석→보고서→이미지→블루프린트→모델샷 자동 흐름을 검증하고, 누락된 기능(보고서 생성/댓글 인사이트/세션별 보고서)을 보강한다.

---

## 0. 전제/제약
- Git 저장소 없음 → 브랜치/PR 불가.
- 파일 ≤ 300 LOC, 함수 ≤ 50 LOC, 매개변수 ≤ 5, 순환 복잡도 ≤ 10 준수.
- 변경 전 관련 파일 전체 읽기(부분 읽기 금지).
- 테스트는 실제 실행 로그로 증빙.

## 1. 핵심 구성 요소/기대 기능
- 크롤링: YouTube(댓글+STT), NateNews(댓글) + SearXNG(뉴스 스니펫).
- 분석: 수집 데이터+숨은 의견(댓글) 기반 AI 트렌드/인사이트.
- 보고서: 분석 결과를 보고서 모델로 생성·저장(언어별), 세션 단위 조회.
- 이미지/블루프린트: AI 프롬프트 → 의상 이미지 → 모델 착장 → 블루프린트(3종).
- 프론트: 개요 로그, 분석 보고서 탭, 이미지 탭 정상 표시.

## 2. 엣지 케이스
- 댓글 데이터 없음/길이 부족, STT 실패, 키워드 추출 실패.
- 보고서 생성 실패/번역 실패.
- 블루프린트 생성 실패(모델/API 키).
- 크롤링 결과 0건.

## 3. 구현/검증 단계
1) 코드/로그 정합성 검토(파이프라인·보고서·댓글 흐름).
2) 보고서 생성 서비스 추가 및 세션별 보고서 저장/조회 연동.
3) 댓글 인사이트 요약을 분석 입력 컨텍스트에 포함.
4) UI 파라미터 보강(보고서 조회에 세션 식별자 반영).
5) 실제 파이프라인 테스트(최소 수집, 블루프린트 포함), 정적 검증 3회+전체 5회.

## 4. 완료 기준
- 세션 단위 보고서 생성/조회 정상.
- 분석 입력에 댓글/숨은 의견이 포함됨.
- 파이프라인 완전 자동 실행 및 결과 로그/이미지/블루프린트 확인.
