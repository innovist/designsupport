## 1. 핵심 구성 요소 점검 및 기대 기능
- 보고서 조회 시 요청 언어로 반환(언어 불일치 시 AI 번역 호출).
- 번역된 보고서는 언어별로 DB 저장(동일 언어 재조회 시 재번역 금지).
- 착장 이미지 생성 시 모델 선택 가능 + GPU 미탑재 환경에서 z-image 제외.

## 2. 기술 문서 리서치/검색 범위
- 보고서 모델/라우팅: `app/models/report.py`, `app/api/routes.py`
- 이미지 생성 로직: `app/services/image_generation_service.py`, `app/services/pipeline_orchestrator.py`
- 시스템 감지: `app/utils/system_detector.py`
- AI 클라이언트: `ai_clients/gemini_client.py`

## 3. 엣지 케이스
- 보고서가 존재하지 않음 → 404.
- 요청 언어가 미지원 → 400.
- 번역 중 AI 실패 → 오류 반환(조용한 실패 금지).
- GPU 미탑재/키 미설정 → z-image 제외.

## 4. 해결 전략
- 보고서 조회 API에서 언어별 존재 여부 확인 → 없으면 번역 후 새 레코드 저장.
- 번역은 Gemini로만 수행(키 미설정 시 명시적 오류).
- 이미지 모델 선택은 하드웨어 조건( GPU + API 키 )을 기반으로 필터링.

## 5. 구현 단계(백엔드/프론트 동시 검토)
1) 보고서 API/서비스 신규 추가(조회 시 번역 저장 포함)
2) 이미지 생성 모델 선택 로직에 GPU 조건 반영
3) 세션 파이프라인에서 착장 이미지 모델 선호값 전달
4) 정적 검증 3회 실행 및 결과 기록

## 6. 테스트 계획

- 착장 이미지 모델 선택 경로 검증(로그 및 모델 선택 결과 확인)

## 7. 완료 기준

## 8. 대안 비교(결정 기록)
1) 보고서 번역을 조회 시점에 수행  
   - 장점: 최신 언어 설정 반영, 요청 시점 제어  
   - 단점: 최초 요청 지연  
   - 위험: AI 호출 실패 시 조회 실패
