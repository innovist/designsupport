## 0. 전제/제약
- 파일 ≤ 300 LOC, 함수 ≤ 50 LOC, 매개변수 ≤ 5, 순환 복잡도 ≤ 10.
- 변경 전 관련 파일 전체 읽기(부분 읽기 금지).
- 실제 실행 로그 기반 테스트 필수.
- Git 저장소 없음 → 브랜치/PR 불가.

## 1. 핵심 구성 요소/기대 기능
- 세션 상태: 실행 중 세션이 워커 종료/재시작 시 실패로 동기화되어야 함.
- 크롤링: YouTube(STT) + NateNews + SearXNG 선택 시 안정적으로 완료/로그 표시.
- 보고서: 조회 언어가 다르면 번역 → 언어별 DB 저장 → 재조회 시 캐시 사용.
- UI: 개요 탭에 키워드/진행률/터미널 로그 명확히 표시.

## 2. 엣지 케이스
- 서버 재시작으로 파이프라인 작업 유실.
- 크롤러 타임아웃으로 수집 0건.
- 보고서 번역 실패/키 누락.
- SearXNG 미설정 상태에서 선택.

## 3. 구현/검증 단계
1) 세션 메타데이터/로그 기반 정체 원인 확인.
2) 실행 중 세션의 워커 상태 동기화(실행 중인데 워커 없음 → 실패 처리/로그 기록).
3) 크롤링 타임아웃 로직 재검토 및 설정값/기본값 합리화.
4) 보고서 조회 언어 번역/DB 캐싱 동작 실제 호출로 검증.
5) 최소 크롤링 파이프라인 실제 실행(YouTube+NateNews, 옵션 최소) 및 로그 캡처.

## 4. 완료 기준
- 정체 세션이 실패로 전환되고 원인이 로그에 노출됨.
- 크롤링/보고서 조회가 실제 실행 로그로 검증됨.
- 보고서 언어별 저장이 확인됨.
- UI 로그/키워드/진행률 표시가 사용자 혼란 없이 해석 가능.
