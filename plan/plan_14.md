# Plan 14: 보고서 언어 기준/정적 검증/이미지·블루프린트 파이프라인 검증

**작성일**: 2025-12-22 21:37  
**목적**: 보고서가 생성 시점의 언어 설정을 따르도록 보장하고, 전체 프로세스 정적 검증 및 의류 이미지·착장 이미지·블루프린트 생성 파이프라인(프롬프트/조건/지침)을 완전 검증한다.

---

## 0. 전제/제약
- Git 저장소가 아님 → 브랜치/PR 단계는 수행 불가(문서로 사유 기록).
- 스펙 문서는 본 계획서(plan_14.md)로 대체.
- 모든 변경 전 관련 파일 전체 읽기(부분 읽기 금지).

## 1. 핵심 구성 요소 점검 및 기대 기능 목록
- 보고서 생성 시 **현재 언어 설정**을 반영해 내용/라벨/문구가 생성됨.
- 전체 UI(페이지/버튼/모달/기능)가 언어 변경 시 완전 번역 유지.
- 의류 이미지/착장 이미지/블루프린트 생성 파이프라인의 프롬프트, 조건, 지침이 최신 코드/레퍼런스와 정합.

## 2. 기술 문서 리서치 및 검색 대상
- 코드: `app/services/*`, `app/api/*`, `templates/pages/*`, `static/js/*`, `static/i18n/*`
- 레퍼런스: `reference/*` 내 관련 문서/코드(필요 범위만)

## 3. 엣지 케이스 도출
- 보고서 생성 시 언어 설정 값 누락/비정상(기본값 처리).
- 보고서 생성 직전 언어 변경 → 캐시/상태 반영 지연.
- 파이프라인 프롬프트/조건이 설정값(사이즈 시스템, 모델 선택 등)과 불일치.
- 이미지/블루프린트 생성 결과 구조 불일치(키 누락/타입 불일치).

## 4. 엣지 케이스 해결 리서치
- 기존 설정 저장/로드 흐름과 보고서 생성 흐름 비교.
- 파이프라인 단계별 입력/출력 스키마 확인.

## 5. 브랜치/스펙 문서
- Git 저장소 부재로 브랜치/PR 생성 불가.
- 스펙 문서: 본 plan_14.md로 대체.

## 6. 스펙에 따른 구현 방향 리서치
- 보고서 생성 경로에서 언어 설정 전달/참조 지점 확인.
- i18n 적용 누락 문자열 정적 스캔/검증.
- 이미지/착장/블루프린트 관련 서비스/프롬프트 파일 전체 검토.

## 7. 구현 단계(백엔드/프론트 동시 검토)
1) 관련 파일 전체 읽기 및 호출/참조 경로 확인  
2) 보고서 언어 기준 검증 및 필요 시 최소 수정  
3) i18n 누락 여부 정적 점검 및 필요 시 보완  
4) 이미지/착장/블루프린트 파이프라인(프롬프트/조건/지침) 검증 보고  
5) 정적 검증 3회 수행 및 결과 기록  
6) worksheet/todo 업데이트

## 8. 테스트/정적 검증 계획
- 전체 프로세스 정적 검증 스크립트 3회 실행(실제 로그 확보).
- 변경 시 관련 경로 기능 테스트(실제 실행).

## 9. 완료 기준
- 보고서가 생성 시점 언어 설정을 정확히 반영함을 코드로 확인 또는 수정 완료.
- 정적 검증 3회 결과 로그 확보.
- 파이프라인 프롬프트/조건/지침 검증 결과 문서화.

## 10. 리스크/대응
- 보고서 경로가 다수일 경우 누락 가능 → 전역 검색 및 호출 체인 추적.
- 프롬프트 템플릿 분산 → 참조 경로 전체 매핑 후 검증.
