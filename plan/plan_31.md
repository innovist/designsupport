# Plan 31: 판매용 소개서 및 제안서 산출물 작성

**작성일**: 2026-03-10 23:45 KST
**목적**: 코드베이스와 실제 UI 스크린샷을 근거로 Fashion Image Gen 시스템의 가치와 판매 포인트를 정리한 소개서/제안서 및 PPT를 생성한다.

---

## 1. 범위
- 입력: 코드베이스, 기존 소개 문서, `marketing_docs` 스크린샷 7장
- 출력: `marketing_docs` 내 판매형 소개서/제안서 markdown, PPT 슬라이드 문서
- 제외: 기능 코드 수정, 모델 교체, 데이터베이스 변경

---

## 2. 검토 기준
- 제품 컨셉: 패션 데이터 수집부터 분석, 디자인 이미지, 블루프린트까지 연결된 운영형 플랫폼
- 판매 관점: SaaS/온프레미스 도입 가치, 확장성, 운영 편의성, 결과 자산화
- 표현 원칙: 실제 구현 범위만 사용, 과장 수치 배제, 스크린샷과 기능 설명 정합 유지

---

## 3. 작업 절차
1. 핵심 코드/문서/테스트/화면을 근거로 제품 의도와 장점 정리
2. 도입 배경, 핵심 기능, 차별점, 적용 방식, 판매 모델을 포함한 markdown 작성
3. 기존 PPT 생성 방식 검토 후 스크린샷을 배치한 새 PPT 산출물 생성
4. 문서와 PPT의 메시지/슬라이드 흐름/파일 생성 여부 검수

---

## 4. 품질 기준
- 코드 구조와 실제 UI를 근거로 한 사실 기반 문안일 것
- 구매자 관점에서 SaaS/온프레미스 비교와 도입 시나리오가 명확할 것
- PPT는 텍스트 중심이되 최소 3장 이상에 실제 스크린샷이 반영될 것
