# Plan 32: 범용 디자인 창작 지원도구 기획 문서 작성

**작성일**: 2026-05-07 17:26 KST
**목적**: 패션 중심 시스템을 산업디자인, 패션디자인, 시각디자인, 광고디자인까지 확장하는 SaaS형 디자인 창작 지원도구로 재기획하고 `User_Needs_v01.md`에 정리한다.

---

## 1. 범위
- 입력: 사용자 신규 방향성, 기존 `user_needs.md`, `reference_library.md`, 현재 구현 구조
- 출력: 루트 `User_Needs_v01.md`
- 포함: 제품 방향, 워크플로우, 벤치마크/레퍼런스 검토, 차별화, 와이어프레임, 클린 아키텍처, 모듈 구성, ERD, 시퀀스 다이어그램, Django/PostgreSQL/Vanilla UI 기반 구현 방향
- 제외: 실제 코드 구현, DB 마이그레이션, AI 모델 변경

---

## 2. 검토 기준
- 편집툴이 아닌 디자인 창작 아이디어 지원도구로 정의한다.
- 레퍼런스 검색, 디자인 트렌드 지식화, 추상화, 컨셉 결정 근거, 스펙 문서화가 중심이어야 한다.
- 자동 진행 모드와 챗봇 협업 모드를 모두 지원한다.
- SaaS 사용자 워크스페이스와 관리자 프로그램을 분리한다.
- 기능별 모듈은 클린 아키텍처와 Repository 패턴을 전제로 한다.

---

## 3. 작업 절차
1. 기존 요구사항과 참고 라이브러리 파일 확인
2. 주요 외부 레포지토리/라이브러리 URL 접근 및 적용 가능성 분류
3. 디자인 도메인 확장 워크플로우와 도메인별 산출물 정의
4. SaaS 아키텍처, 모듈 구조, ERD, 시퀀스 다이어그램 설계
5. `User_Needs_v01.md` 작성 및 검토

---

## 4. 품질 기준
- 추측성 구현 주장 없이 확인한 자료와 논리적 판단을 구분한다.
- 가져올 수 있는 라이브러리와 참고만 할 자료를 명확히 분류한다.
- MVP 범위와 장기 확장 범위를 분리한다.
- 문서는 이후 구현 계획의 기준으로 사용할 수 있을 정도로 구조화한다.
