# Plan 20: STT 항상 활성화 정책 반영

**작성일**: 2025-12-23 03:28  
**목적**: STT는 사용자가 선택하지 않고 항상 활성화되도록 UI/파이프라인을 단순화하고, 기존 세션에도 기본값이 적용되도록 한다.

---

## 0. 전제/제약
- Git 저장소 부재로 브랜치/PR 단계 수행 불가.
- 파일 ≤ 300 LOC, 함수 ≤ 50 LOC, 매개변수 ≤ 5, 순환 복잡도 ≤ 10 준수.
- 변경 전 관련 파일 전체 읽기(부분 읽기 금지).
- 분리 작업은 복잡도를 높이지 않도록 최소화.

## 1. 핵심 구성 요소 및 기대 기능
- 세션 생성 UI에서 STT 선택 제거(항상 사용).
- 세션 생성 payload에는 STT 활성화 값이 항상 반영.
- 파이프라인은 STT 설정이 비어 있어도 기본 True로 동작.

## 2. 대안 비교(결정 기록)
1) UI만 STT 항상 True로 고정
   - 장점: 변경 최소, UX 단순
   - 단점: 외부 API가 false를 보내면 비활성화 가능
   - 위험: 우회 경로에서 정책 불일치
2) 파이프라인 기본값을 True로 강제
   - 장점: 모든 실행 경로에서 정책 일관
   - 단점: STT 비활성화 옵션 제거
   - 위험: 처리 시간 증가(의도된 정책)

→ 선택: 2) 파이프라인 기본값 True + UI 고정

## 3. 구현 단계(백엔드/프론트 동시)
1) 세션 모달에서 STT 토글 제거, 설명 문구만 유지.
2) 세션 생성 JS에서 STT 값을 항상 true로 전송.
3) 크롤링 유틸에서 STT 기본값 True 적용.

## 4. 테스트 계획
- 정적 검증 3회 실행.
- 전체 정적 검증 5회 실행.
- (필요 시) YouTube STT 경로 최소 1회 확인.

## 5. 완료 기준
- UI에서 STT를 사용자가 선택하지 않아도 항상 활성화.
- 파이프라인 실행 시 STT 설정 누락 시에도 활성화.
- 정적 검증 기준 충족.
