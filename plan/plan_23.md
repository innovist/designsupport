# Plan 23: 크롤링 정합성/로그/진행률 보강 및 보고서 언어 조회 검증

**작성일**: 2025-12-23 05:25  
**목적**: 크롤링 무한 대기·로그 불명확·진행률 혼란을 해소하고, 보고서 조회 시 언어 번역/캐시 저장을 검증한다. SearXNG 목적 반영과 전체 자동 파이프라인(크롤링→분석→보고서→이미지→블루프린트→모델샷) 흐름을 실제 테스트로 확인한다.

---

## 0. 전제/제약
- Git 저장소 없음 → 브랜치/PR 불가.
- 파일 ≤ 300 LOC, 함수 ≤ 50 LOC, 매개변수 ≤ 5, 순환 복잡도 ≤ 10.
- 변경 전 관련 파일 전체 읽기(부분 읽기 금지).
- 실제 실행 로그 기반 테스트 필수.

## 1. 핵심 구성 요소/기대 기능
- 크롤링: YouTube(STT), NateNews, SearXNG 동작/로그/진행률 명확화.
- 분석: 패션 트렌드 목적의 키워드/데이터 정합성 유지.
- 보고서: 조회 시점 언어 출력, 번역 결과 언어별 DB 저장.
- 이미지/블루프린트: 프롬프트·조건·지침 검증 및 파이프라인 완전 자동 실행.
- 프론트: 개요 탭에 크롤링 로그/키워드/진행률 설명 표시.

## 2. 엣지 케이스
- 크롤러 타임아웃/무한 대기.
- 설정된 수집량과 진행률 표시 불일치.
- 보고서 언어 미존재/번역 실패.
- YouTube STT 실패/부분 실패.
- SearXNG 미설정 선택.

## 3. 구현/검증 단계
1) DB/로그로 최근 실패 세션 원인 확인.
2) 크롤러 타임아웃 처리 및 실패 로그 가시화 보강.
3) 진행률 산정 로직 정밀화 및 개요 탭 설명 문구 추가.
4) 보고서 조회 언어 번역/캐시 저장 동작 확인.
5) 실제 파이프라인 최소 크롤링 테스트(실패 세션 옵션 기반), 정적 검증 3회+전체 5회.

## 4. 완료 기준
- 크롤링 단계가 무한 대기 없이 종료되고 오류 원인이 로그에 표시됨.
- 진행률/수집량 설명이 사용자 혼란 없이 해석 가능.
- 보고서 조회 언어 번역 및 언어별 저장 확인.
- 자동 파이프라인 전체 실제 실행 로그 확보.
