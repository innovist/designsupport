# Plan 16: 보고서 조회 UX + GPU 기반 모델 선택 UX + 대시보드 분리

**작성일**: 2025-12-22 23:21  
**목적**: 분석 보고서 탭이 조회 시점 언어로 보고서를 표시하도록 연결하고, 프로젝트 이미지 모델 선택 UI를 GPU(NVIDIA) 조건에 따라 활성/비활성 처리한다. 대시보드 파일은 LOC 제한을 준수하도록 분리한다.

---

## 0. 전제/제약
- Git 저장소 부재로 브랜치/PR 단계 수행 불가.
- 파일 ≤ 300 LOC, 함수 ≤ 50 LOC, 매개변수 ≤ 5, 순환 복잡도 ≤ 10 준수.
- 모든 변경 전 관련 파일 전체 읽기.
- 분리 작업이 복잡도를 높이지 않도록 구조 단순 유지.

## 1. 핵심 구성 요소 및 기대 기능
- 분석 보고서 탭: 조회 시점 언어로 `/api/v1/reports` 호출 → 없으면 안내 메시지 표시.
- 언어 변경 시 보고서 재조회(번역 캐시 활용).
- 프로젝트 생성/수정 모달에 이미지 모델 선택 UI 추가.
- NVIDIA GPU 미탑재 시 z-image 선택 비활성화(클라우드 모델은 항상 선택 가능).
- 대시보드 템플릿/스크립트 LOC 제한 준수.

## 2. 기술 문서 리서치/검색 범위
- i18n: `static/js/i18n.js`, `static/i18n/*.json`
- 대시보드 UI: `templates/pages/dashboard.html`, `static/js/pages/dashboard.js`
- 보고서 API: `app/api/reports.py`
- 시스템 감지: `app/utils/system_detector.py`
- 설정 API: `app/api/settings_ui.py`

## 3. 엣지 케이스
- 보고서 미존재(404) → “보고서 없음” 메시지.
- 세션 미완료 → “완료 후 조회 가능” 안내.
- 번역 실패(500) → 오류 메시지 표시.
- GPU 비활성/미탑재 → z-image 비활성화, 기존 선택값 유지 안내.

## 4. 해결 전략
- 설정 UI에 GPU 상태/모델 가용성 엔드포인트 추가.
- 대시보드에서 GPU 상태를 로드해 z-image 옵션 disabled 처리.
- 보고서 탭 진입 시 또는 언어 변경 시 보고서 재조회.
- 대시보드 HTML 모달 분리, JS 모듈 분할로 LOC 준수.

## 5. 구현 단계(백엔드/프론트 동시)
1) `system_detector.py` GPU 감지 규칙 정비(NVIDIA 기준).
2) `settings_ui.py`에 image-models 상태 엔드포인트 추가.
3) 대시보드 HTML 모달 분리 + 스크립트 모듈 분할.
4) 보고서 탭 렌더링 및 언어 변경 재조회 로직 구현.
5) 프로젝트 생성/수정에 모델 선택 UI 및 payload 연결.
6) i18n 키 추가(ko/en/zh-CN/zh-TW).
7) 정적 검증 3회 실행 및 기록.

## 6. 테스트 계획
- 정적 검증 3회 실행 로그 확보.
- 대시보드 UI에서:
  - 프로젝트 생성/수정 시 모델 값 전달 확인
  - GPU 미탑재 시 z-image 비활성화 표시 확인
  - 언어 변경 시 보고서 재조회 동작 확인

## 7. 완료 기준
- 보고서 탭이 조회 언어 기준으로 렌더링.
- z-image 옵션이 NVIDIA GPU 유무에 따라 활성/비활성.
- 대시보드 파일 LOC 제한 준수.
