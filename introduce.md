# 패션 트렌드 분석 및 AI 디자인·블루프린트 생성 시스템 소개서

## 1. 문서 목적
이 문서는 **Fashion Image Gen 시스템**을 외부 조직/파트너에게 소개하기 위한 기초 자료입니다. 기술적 구조와 비즈니스 가치를 동시에 설명하며, 추후 제안서·기획서·영업자료의 근간으로 사용할 수 있도록 상세하게 정리합니다.

---

## 2. 시스템 개요
본 시스템은 **패션 트렌드 데이터 수집 → AI 기반 분석 → 디자인 아이디어 및 이미지 생성 → 블루프린트(도면) 생성**까지 이어지는 **엔드-투-엔드 패션 기획 자동화 플랫폼**입니다. 웹 기반 대시보드와 API를 함께 제공하며, 프로젝트/세션 단위로 파이프라인을 실행하고 결과를 축적합니다.

---

## 3. 핵심 가치
- **데이터 기반 의사결정**: 커뮤니티/플랫폼/미디어/검색 데이터를 모아 객관적 트렌드 신호를 수집합니다.
- **분석→기획→생성의 자동화**: 분석 결과가 곧바로 디자인 아이디어/이미지/도면으로 이어지는 일관된 워크플로우를 제공합니다.
- **프로젝트 단위 자산 축적**: 세션 결과(리포트, 이미지, 블루프린트)를 프로젝트 단위로 저장·관리합니다.
- **확장 가능한 구조**: 크롤러, AI 모델, 이미지 생성 모델을 모듈 형태로 확장할 수 있습니다.

---

## 4. 시스템 아키텍처

### 4.1 전체 구성
- **Frontend**: Jinja2 템플릿 + Vanilla JS + CSS
- **Backend**: FastAPI (Python), 비동기 파이프라인 오케스트레이션
- **AI 계층**: Gemini, GLM (텍스트 분석/보고서), Seedream·Nano Banana·Z-Image (이미지 생성)
- **데이터 계층**: SQLAlchemy + SQLite(기본), 설정/결과 파일 저장
- **보조 서비스**: SearXNG 로컬 검색, GPU 감지

### 4.2 Backend 주요 모듈
- `app/services/pipeline_orchestrator.py`: 전체 파이프라인 실행/조율
- `app/services/analysis_service.py`: 다중 모델 트렌드 분석 및 종합
- `app/services/report_generation_service.py`: 리포트 JSON 생성/저장
- `app/services/image_generation_service.py`: 이미지 모델 선택 및 생성
- `app/services/blueprint_service.py`: 스케치/레이아웃/패턴 생성
- `crawlers/crawler_service.py`: 크롤러 오케스트레이션
- `app/api/session_store.py`: 세션 상태·로그·진행률 관리

### 4.3 Frontend 주요 화면
- **홈**: 빠른 시작/최근 결과/상태 요약
- **대시보드**: 프로젝트/세션 관리, 탭 기반 결과 열람
- **세션 상세**: 진행률/로그/요약/결과 확인
- **라이브러리**: 생성 이미지 필터링/다운로드
- **설정**: API 키, 크롤러, 모델, AI 조사, 유튜브 채널 관리
- **챗봇**: 프로젝트 맥락 기반 질의응답

### 4.4 데이터 저장 구조
- DB: `storage/fashion.db` (SQLite 기본)
- 설정: `storage/config/user_settings.json`
- 업로드: `uploads/`
- 로그: `logs/`
- 결과/참조: `storage/results`, `storage/references`

### 4.5 아키텍처 상세 (레이어 관점)
- **UI/Presentation Layer**: Jinja2 템플릿 기반 UI, 대시보드/세션/라이브러리/설정/챗봇 페이지 제공
- **API Layer**: `/api/v1/*` 라우트로 세션/프로젝트/설정/크롤러/리포트/라이브러리 제공
- **Orchestration Layer**: 파이프라인 단계별 진행률, 로그, 상태 관리
- **AI Service Layer**: 텍스트 분석(클라우드 LLM), 이미지/도면 생성(멀티 모델)
- **Crawler Layer**: 키워드 기반 병렬 수집, 유튜브 채널 수집, SearXNG 연동
- **Storage Layer**: DB(프로젝트/세션/리포트), 파일(이미지/업로드/로그/설정)

### 4.6 실행 및 네트워크 구성
- **웹 앱 기본 포트**: `http://localhost:8912`
- **API 문서**: `http://localhost:8912/docs`
- **SearXNG(옵션)**: `http://localhost:8913` (설정 페이지에서 URL 지정)

## 5. 파이프라인 (8단계)

1. **입력 분석**
   - 세션 설명/추가 입력/URL/파일(PDF/이미지)을 통합하여 컨텍스트를 구성합니다.
   - 이미지 파일은 Gemini로 요약 설명을 생성합니다.

2. **키워드 추출**
   - Gemini로 검색 키워드를 생성하고, 실패 시 GLM 폴백으로 전환합니다.
   - 정확히 5개 키워드를 유지하며 3단어 이내로 정제합니다.

3. **크롤링 수집**
   - 키워드별로 다중 크롤러를 병렬 실행합니다.
   - 날짜 범위, 소스당 최대 수집량, 유튜브 채널 설정을 반영합니다.

3.5. **AI 직접 조사(선택)**
   - 설정에서 활성화 시 Gemini Search / Perplexity / GLM 웹리서치를 병렬 수행합니다.
   - 결과는 세션 결과에 포함됩니다.

4. **트렌드 분석**
   - 수집 데이터 + 댓글 인사이트를 전처리하여 분석합니다.
   - Gemini 2.5 Flash, Gemini 2.5 Pro, GLM-4.7 결과를 종합합니다.
   - 대상 필터(연령/성별/카테고리/시즌) 제약을 명시해 분석합니다.

5. **디자인 아이디어 생성**
   - 종합 분석 결과에서 컨셉을 추출하여 3~5개 아이디어를 도출합니다.
   - 각 아이디어는 이미지 생성용 프롬프트를 포함합니다.

6. **리포트 생성**
   - 분석/아이디어/데이터 통계를 종합해 JSON 리포트를 생성합니다.
   - 다국어(ko/en/zh-CN/zh-TW) 리포트를 지원합니다.

7. **이미지 생성**
   - 마스터 디자인(Flat Lay) → 모델 착장 이미지 순으로 생성합니다.
   - Z-Image/Seedream/Nano Banana 모델을 상황에 따라 선택합니다.

8. **블루프린트 생성**
   - 스케치/레이아웃/패턴 3종을 생성합니다.
   - KS/GB/ASTM/ISO 치수 체계를 지원합니다.

## 6. 워크플로우 (사용자 관점)

1. **설정**
   - API 키(Gemini/GLM/Seedream/Nano Banana/Perplexity) 등록
   - 크롤러 동시 작업 수, 타임아웃, SearXNG URL 설정
   - 필요 시 AI 직접 조사 기능 활성화

2. **프로젝트 생성**
   - 브랜드/라인/시즌 단위로 프로젝트를 생성합니다.

3. **세션 생성**
   - 입력 문장, URL, 파일, 필터(연령/성별/카테고리/시즌)를 설정합니다.
   - 크롤러 소스, 수집 기간, 유튜브 채널 옵션을 선택합니다.

4. **파이프라인 실행**
   - 자동 시작 또는 수동 시작
   - 진행률/로그 실시간 확인

5. **결과 활용**
   - 대시보드: 요약/데이터/분석/이미지/블루프린트 열람
   - 라이브러리: 생성 이미지 필터링 및 다운로드
   - 리포트: JSON 구조로 추출 가능

### 6.1 사용 방법 (실행/운영 가이드)

#### 6.1.1 실행
- 기본 실행: `python main.py`
- 개발 실행: `uvicorn main:app --reload --host 0.0.0.0 --port 8912`
- 기본 접속: `http://localhost:8912`

#### 6.1.2 초기 설정
1. 설정 페이지 접속 → API 키 등록 (Gemini/GLM/Seedream/Nano Banana/Perplexity)
2. 크롤러 동시 실행 수/타임아웃/SearXNG URL 설정
3. 이미지 모델 및 AI 조사 옵션 설정(필요 시)

#### 6.1.3 프로젝트/세션 생성
1. 프로젝트 생성(브랜드/라인/시즌 단위)
2. 세션 생성(입력/필터/크롤러 설정)
3. 자동 시작 또는 수동 실행

#### 6.1.4 실행 모니터링
- 세션 상세 페이지에서 진행률, 로그, 에러 메시지 확인
- 완료 시 결과 탭에서 분석/이미지/블루프린트 확인

#### 6.1.5 결과 활용
- 라이브러리에서 결과 이미지 필터링/다운로드
- 리포트 JSON으로 외부 문서/보고서 작성 가능

- **패션 플랫폼**: 무신사, 인스타그램, 핀터레스트

- **미디어**: 유튜브
- **뉴스/언론**: 패션N, WGSN, 네이트뉴스

### 7.2 파이프라인 기본 등록 크롤러
파이프라인 실행 기준으로 **기본 등록된 크롤러**는 다음과 같습니다.

- 추가로 **SearXNG**가 자동 등록됩니다.

> 주의: 카탈로그에 존재하더라도, CrawlerService에 등록되지 않은 크롤러는 실행되지 않습니다. 확장 시 등록만 추가하면 바로 연동 가능합니다.

## 13. 현재 구현 범위 및 유의사항
- AI 직접 조사 기능은 **설정에서 활성화 시 결과를 수집**하지만, 분석 단계에 전달되는 통합 흐름은 **오케스트레이터에서 기본 분석 호출로 실행**됩니다.
- 챗봇 대화 기록은 **인메모리 저장소**를 사용하므로 서버 재시작 시 초기화됩니다.
- API 키는 `storage/config/user_settings.json`에 저장되며, 배포 시 보안 정책에 따라 관리가 필요합니다.
- Z-Image는 GPU 사용 여부에 따라 활성/비활성됩니다.

## 14. 확장 포인트
- **크롤러 추가**: 신규 크롤러를 `CrawlerService.register_crawler()`로 등록
- **AI 모델 확장**: settings_storage의 모델 목록 확장 및 클라이언트 추가
- **분석 강화**: ai_research → analysis 통합 경로 연결 강화 가능
- **데이터베이스 확장**: SQLite → PostgreSQL 전환 가능
